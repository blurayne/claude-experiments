# Slothy-Hutty

A fully procedural 3D model of **Slothy** — the sloth-faced, Jabba-shaped diner mascot from the
concept sheet (cream cap with a red band, droopy panda eyes, stubby arms, a tapering slug tail,
and a **SLOTHY'S** burger apron). The same character is rendered by two hand-written engines and
can be exported for 3D printing.

> The published page at `/slothy-hutty/` is this document. Open one of the viewers below.

## Viewers

- [`webgl2.html`](webgl2.html) — real-time renderer written directly against **WebGL2**
  (GLSL ES 300).
- [`webgpu.html`](webgpu.html) — the same scene written directly against **WebGPU** (WGSL).
  Needs a WebGPU-capable browser (recent Chrome/Edge, Safari 18+, or Firefox with WebGPU on);
  it falls back to a message linking to the WebGL2 page otherwise.

Both viewers share one procedural model and offer the same controls:

- **drag** to orbit, **scroll** to zoom, **Auto-rotate** / **Reset view**
- **Face** button cycles all three expressions from the sheet — *Smug* (closed), *Sleepy*
  (half-open with a drool drip), *Laughing* (wide open with teeth + tongue)
- **Shading** button toggles **Smooth** (Blinn-Phong on a studio backdrop) and **Cel**
  (toon bands + inked outline on a paper backdrop)
- **Download** menu exports the current pose in every 3D-print format below

No external libraries or CDNs — the engines, math, geometry, and exporters are all hand-written
and run offline.

## 3D-print downloads

The committed files (regenerated by `build.js`) are the closed/smug pose, scaled to ~110 mm and
oriented Z-up on the bed:

- [`slothy-hutty.stl`](slothy-hutty.stl) — plain geometry, single-material printing.
- [`slothy-hutty-color.stl`](slothy-hutty-color.stl) — colored binary STL (Materialise
  per-facet RGB extension).
- [`slothy-hutty.3mf`](slothy-hutty.3mf) — colored 3MF (per-region base materials); the standard
  for multicolor FDM (Bambu / Prusa / Cura).
- [`slothy-hutty.obj`](slothy-hutty.obj) + [`slothy-hutty.mtl`](slothy-hutty.mtl) +
  [`slothy-hutty.png`](slothy-hutty.png) — OBJ with materials and a UV-mapped apron texture.

> STL cannot store textures or color, so the "with texture/color" request is delivered through
> the colored STL, the 3MF, and the textured OBJ; the plain STL is the geometry-only version.

## Files

- `slothy-math.js` — minimal mat4/vec3 helpers (WebGL and WebGPU clip-space projections).
- `slothy-model.js` — procedural geometry: the fat fused body (torso, head, jowls, arms, feet,
  tail) is one organic mesh built from a **metaball signed-distance field via naive Surface
  Nets**; facial features (hooded eyes, brow, snout), three mouth variants, cap, and a
  body-conforming apron shell are placed on top, with per-vertex colors and the apron art
  (`paintApron`).
- `slothy-export.js` — painters, a dependency-free PNG encoder, a store-only ZIP writer, and the
  STL / 3MF / OBJ serializers.
- `slothy-ui.js` — shared orbit camera, HUD, animation loop, and download wiring.
- `build.js` — regenerates the print artifacts from the model (`node build.js`).

## Running locally

ES-module imports need an HTTP origin (not `file://`):

```bash
cd slothy-hutty
python3 -m http.server 8000   # then open http://localhost:8000/webgl2.html
node build.js                 # regenerate the 3D-print files
```
