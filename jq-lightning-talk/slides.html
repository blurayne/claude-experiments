What is jq?

  • "sed / awk for JSON" — and then some.
  • A filter pipeline: every filter takes one value, emits zero or more.
  • Stephen Dolan, 2012, C. Ships almost everywhere.

$ curl -s api/users | jq '
    .users[]
    | select(.active)
    | {id, email, n: (.posts|length)}
  '

One input → a stream of outputs. | chains. That's the whole language.

It's a real language surprise

Variables · functions · reduce / foreach · if/then · try/catch · modules · recursive descent ..


def factorial:
  if . <= 1 then 1
  else . * (. - 1 | factorial)
  end;

10 | factorial          # => 3628800

Pure, functional, point-free. Recursion just works.

jq vs JMESPath grammar

JMESPath

AWS CLI --query, Azure CLI, Splunk…


users[?active].{
  id: id,
  email: email
}
  • Declarative query language
  • No recursion · no user functions
  • Not Turing-complete (by design)
  • Trivial to embed & sandbox

jq

Same query:


.users[]
| select(.active)
| {id, email}
  • Programming language
  • Fns, recursion, generators
  • Turing-complete
  • Pays for it in surface area

Grammar-wise jq is a PEG-shaped pipeline (ordered sequencing, alternation, no ambiguity) — that's why filters compose so cleanly. JMESPath is just a closed expression grammar over JSON.

Turing-complete yes, really

  • Recursive functions ✓
  • Conditional branching ✓
  • Unbounded data (arrays, objects) ✓
  • Generators ⇒ arbitrary iteration ✓

People have written: brainfuck interpreters, lambda-calculus evaluators, Game of Life, even a TCP state machine — in jq.

Should you?  probably not.

Proof-by-construction

A while combinator over a generator is enough:


def while(cond; update):
  def w: if cond then ., (update | w) else empty end;
  w;

# count 0..9
0 | while(. < 10; . + 1)

With while + recursion + arrays as a tape → you can simulate a Turing machine. QED, and please look away.

The jq family

namelangangle
jq C the original
gojq Go pure-Go port, library-friendly
jaq Rust fast, more "correct"
fq Go jq for binary formats
faq Go jq for any format

Same language, very different super-powers.

Faster & embeddable: gojq + jaq

gojq

  • Pure Go, no CGO
  • Library for Go tools (powers fq)
  • Nicer errors, big-decimal numbers

q, _ := gojq.Parse(".users[] | .email")
iter := q.Run(input)
for { v, ok := iter.Next(); if !ok { break }; … }

jaq

  • Rust, ~2–10× faster than jq
  • Fixes some jq quirks
  • Drop-in for most queries

$ jaq '.[] | select(.score > 0.9) | .id' \
    results.json

fq — jq for binary wader/fq

PNG, ELF, MP4, protobuf, PCAP, gzip, … all decoded into a queryable tree.


$ fq '.chunks[] | select(.type == "IDAT") | .len' cat.png
8192
8192
4017

$ fq -d mp4 '.tracks[].media.type' video.mp4
"vide"
"soun"

A hex viewer, a parser, and a REPL — all at once.

faq — jq for any format jzelinskie/faq

JSON · YAML · TOML · XML · BSON · INI · CSV/TSV


$ faq '.services | keys' docker-compose.yaml
["db", "web"]

# TOML → JSON, with a transform on the way through
$ faq -o json '.dependencies | to_entries' Cargo.toml

"the JSON is actually YAML." it always is.

tl;dr

  • jq is a tiny pure-functional language hiding in a CLI.
  • It's Turing-complete — JMESPath isn't. Pick power vs. predictability.
  • The runtimes split: gojq · jaq · fq · faq.
  • Pick the implementation that matches your input, not your taste.
