What is jq?

  • "sed / awk for JSON" — but actually much more.
  • A filter language: data flows through a pipeline.
  • Originally by Stephen Dolan (2012), C implementation.
  • Ships almost everywhere. curl … | jq . is now muscle memory.

$ echo '{"name":"ada","langs":["en","fr"]}' | jq '.langs[0]'
"en"

It's a pipeline of filters


.users[]                     # iterate users
| select(.active)            # keep active ones
| {id, email, n: (.posts|length)}

Every filter takes one input, emits zero or more outputs.
| chains them. That's basically the whole language.

It's a real language surprise

  • Variables: . as $x | …
  • Functions, including parameters & closures
  • reduce / foreach over generators
  • if / then / else, try / catch, labels & break
  • Recursive descent: ..
  • Modules (import "x" as x;)

Functions & recursion


def factorial:
  if . <= 1 then 1
  else . * (. - 1 | factorial)
  end;

10 | factorial          # => 3628800
        

def fib:
  if . < 2 then .
  else (. - 1 | fib) + (. - 2 | fib)
  end;

Turing-complete yes, really

  • Recursive functions ✓
  • Unbounded data structures (arrays, objects) ✓
  • Conditional branching ✓
  • Generators give you arbitrary iteration ✓

People have written: brainfuck interpreters, lambda-calculus evaluators, a Game of Life, even a TCP-state-machine in jq.

Should you?  …probably not.

A tiny proof-by-construction

Counter machine — increments a register until a predicate holds:


def while(cond; update):
  def w: if cond then ., (update | w) else empty end;
  w;

# count from 0 to 9
0 | while(. < 10; . + 1)

With while + recursion + arrays as tape, you can simulate a Turing machine. That's the whole argument.

The jq family

namelangangle
jq C the original
gojq Go pure-Go port, library-friendly
jaq Rust fast, more "correct"
fq Go jq for binary formats
faq Go jq for any format

Same language, very different super-powers.

gojq itchyny/gojq

  • Pure Go reimplementation — no libjq CGO.
  • Embeddable as a library; the engine for fq and many Go CLIs.
  • Nicer error messages, arbitrary-precision numbers.

query, _ := gojq.Parse(".users[] | select(.active) | .email")
iter := query.Run(input)
for {
    v, ok := iter.Next()
    if !ok { break }
    fmt.Println(v)
}

jaq 01mf02/jaq

  • Rust, often ~2–10× faster than jq.
  • Fixes a handful of jq quirks (sort stability, number precision).
  • Drop-in for most everyday queries.

$ jaq '.[] | select(.score > 0.9) | .id' results.json

fq wader/fq

jq, but the input is binary. PNG, ELF, MP4, protobuf, PCAP, …


$ fq '.chunks[] | select(.type == "IDAT") | .len' cat.png
8192
8192
4017

$ fq -d mp4 '.tracks[].media.type' video.mp4
"vide"
"soun"

It's a hex viewer, a parser, and a REPL — all at once.

faq jzelinskie/faq

"Format-agnostic jq" — same syntax, many encodings:

JSON · YAML · TOML · XML · BSON · INI · CSV/TSV


$ faq '.services | keys' docker-compose.yaml
[
  "db",
  "web"
]

# Convert TOML → JSON with a transform on the way through
$ faq -o json '.dependencies | to_entries' Cargo.toml

Which one do I reach for?

  • jq — default, ubiquitous, "just there".
  • jaq — when jq is the bottleneck.
  • gojq — when you're embedding jq into a Go tool.
  • fq — when the JSON is actually a binary blob.
  • faq — when the JSON is actually YAML. (it always is.)

tl;dr

  • jq is a tiny pure-functional language hiding inside a CLI.
  • It is Turing-complete — but please don't.
  • The ecosystem branched into gojq / jaq / fq / faq.
  • Pick the implementation that matches your input, not your taste.
