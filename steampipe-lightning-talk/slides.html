What is Steampipe?

  • A Postgres you can SELECT from…
  • …where the tables are actually APIs.
  • Implemented as a Postgres Foreign Data Wrapper + plugins.
  • 140+ plugins: AWS, GCP, Azure, Kubernetes, GitHub, Slack, CSV, Net, …

steampipe plugin install aws kubernetes csv
steampipe query

Kubernetes plugin

Every resource kind becomes a table.


-- Pods that are not Running, across all namespaces
select
  namespace,
  name,
  phase,
  pod_ip,
  age(creation_timestamp) as age
from kubernetes_pod
where phase <> 'Running'
order by age desc;

Uses your current kubeconfig context — no extra glue.

Joins for free


-- Deployments whose images come from a public registry
select
  d.namespace,
  d.name        as deployment,
  c ->> 'image' as image
from
  kubernetes_deployment as d,
  jsonb_array_elements(d.template -> 'spec' -> 'containers') as c
where
  c ->> 'image' not like 'ghcr.io/acme/%';

Plain Postgres — jsonb, CTEs, window functions all work.

Aggregator connections superpower

One virtual connection that fans out across many real ones.


# ~/.steampipe/config/aws.spc
connection "aws_prod"    { plugin = "aws", profile = "prod"    }
connection "aws_staging" { plugin = "aws", profile = "staging" }
connection "aws_dev"     { plugin = "aws", profile = "dev"     }

connection "aws_all" {
  plugin      = "aws"
  type        = "aggregator"
  connections = ["aws_prod", "aws_staging", "aws_dev"]
}
        

-- One query, three accounts
select _ctx ->> 'connection_name' as account,
       count(*) as buckets
from   aws_all.aws_s3_bucket
group  by 1;

AWS plugin


-- Public S3 buckets across all accounts
select
  account_id,
  name,
  region,
  bucket_policy_is_public
from aws_all.aws_s3_bucket
where bucket_policy_is_public;
        

-- EC2 instances missing a "Owner" tag
select account_id, region, instance_id, instance_type
from   aws_all.aws_ec2_instance
where  not (tags ? 'Owner');

JSON output scriptable


steampipe query \
  --output json \
  "select instance_id, instance_type, region
     from aws_all.aws_ec2_instance
     where state = 'running'" \
| jq '.rows | group_by(.region) | map({region: .[0].region, n: length})'

Also: --output csv, table, line. Pipes into anything.

CSV plugin my favourite trick

Drop a .csv in a folder → it's a table.


# ~/.steampipe/config/csv.spc
connection "csv" {
  plugin = "csv"
  paths  = ["~/data/*.csv"]
}
        

-- Join a hand-maintained owners list with live cloud state
select
  i.instance_id,
  i.region,
  o.team,
  o.contact
from   aws_all.aws_ec2_instance      as i
left join csv.owners                 as o
  on o.account_id  = i.account_id
 and o.instance_id = i.instance_id
where  o.team is null;   -- orphans!

End note tangent

CSV-meets-SQL is cute in Steampipe…

…but DuckDB is even cooler for it.


-- DuckDB, zero config
select team, count(*)
from   read_csv_auto('owners.csv')          o
join   read_parquet('s3://logs/2026-*.pq')  l
  on   l.instance_id = o.instance_id
group  by team;

Parquet, S3, JSON, Excel, HTTPS — all as tables. In-process, no server.

tl;dr

  • Steampipe = SQL over your cloud / cluster / SaaS.
  • Aggregator connections = multi-account in one query.
  • --output json → glues nicely into shell pipelines.
  • CSV plugin → join spreadsheets with live infra.
  • For pure file-crunching → reach for DuckDB.
